# Запуск локального сервера для просмотра изменений

## Быстрый старт

### 1. Установи зависимости (если еще не установлены)
```bash
npm install
```

### 2. Настрой переменные окружения
Создай файл `.env.local` в корне проекта:
```bash
cp .env.production .env.local
# Или создай вручную с нужными переменными
```

Минимально необходимые переменные:
```
DATABASE_URL=postgresql://...
```

### 3. Запусти сервер разработки
```bash
npm run dev
```

Сервер запустится на: **http://localhost:3000**

## Команды

### Запуск в режиме разработки
```bash
npm run dev
```
- Автоматическая перезагрузка при изменениях
- Подробные ошибки в консоли
- Hot reload для быстрого просмотра изменений

### Запуск в production режиме (для тестирования)
```bash
npm run build
npm start
```
- Оптимизированная сборка
- Ближе к production окружению

### Остановка сервера
Нажми `Ctrl+C` в терминале

## Просмотр изменений

1. **Открой браузер:** http://localhost:3000
2. **Внеси изменения** в код
3. **Страница автоматически обновится** (Hot Module Replacement)
4. **Проверь консоль браузера** (F12) для отладки

## Отладка

### Просмотр логов сервера
Логи выводятся в терминал, где запущен `npm run dev`

### Просмотр логов в браузере
- Открой DevTools (F12)
- Вкладка "Console" - ошибки JavaScript
- Вкладка "Network" - запросы к API

### Проверка API endpoints
- http://localhost:3000/api/events
- http://localhost:3000/api/locations
- и т.д.

## Проблемы и решения

### Порт 3000 занят
```bash
# Используй другой порт
PORT=3001 npm run dev
```

### Ошибки базы данных
- Проверь `DATABASE_URL` в `.env.local`
- Убедись, что база данных доступна

### Изменения не применяются
- Останови сервер (Ctrl+C)
- Удали `.next` папку: `rm -rf .next`
- Запусти снова: `npm run dev`

## Полезные команды

```bash
# Очистка кеша Next.js
rm -rf .next

# Переустановка зависимостей
rm -rf node_modules package-lock.json
npm install

# Проверка типов TypeScript
npm run type-check  # если есть такая команда
```
